# Propuesta de Mantenimiento del Sistema de Control de Asistencia

El mantenimiento del sistema es esencial para asegurar su continuidad operativa, prevenir fallos y permitir que la aplicación evolucione conforme cambian las necesidades de los usuarios y del entorno tecnológico. Esta propuesta resume el enfoque de mantenimiento recomendado para la aplicación de control de asistencia, considerando sus características actuales y las funcionalidades proyectadas, como el reconocimiento facial.

## Tipos de Mantenimiento Propuestos
### 1. Mantenimiento Correctivo

Se aplicará cuando se identifiquen fallos en el funcionamiento normal del sistema. El objetivo es restaurar la operación sin afectar la integridad de los datos ni interrumpir el servicio más de lo necesario.

Criterio de aplicación:
Se ejecutará de forma reactiva, bajo reportes de usuarios, monitoreo interno o resultados de pruebas.

### 2. Mantenimiento Preventivo

El propósito es anticiparse a problemas antes de que aparezcan. Incluye mejoras internas y ajustes que incrementen la estabilidad del sistema.

Periodicidad:
Se recomienda realizar revisiones trimestrales.

### 3. Mantenimiento Perfectivo

Enfocado en mejorar o expandir el sistema según nuevas demandas o propuestas de valor.

Ejemplos aplicables al proyecto:
- Incorporación de nuevos tipos de notificaciones o indicadores estadísticos.
- Mejoras en la interfaz para hacer más intuitivo el registro de asistencia.
- Extensión del módulo de reportes para permitir exportación avanzada.
- Implementación del reconocimiento facial como método adicional de marcación.


### 4. Mantenimiento Adaptativo

Permite que el software continúe funcionando cuando cambian las condiciones externas del entorno tecnológico o normativo.



Priorización

Las intervenciones se clasificarán según su criticidad:
Alta: errores que bloquean el registro o comprometen seguridad.
Media: problemas que afectan funcionalidades secundarias.
Baja: mejoras estéticas o ajustes no urgentes.
